## 2. Deep Dive: The `data/` Directory
The `data/` folder functions as the absolute core of the project's payload delivery. It is meticulously structured to prevent data loss during long API calls.

- **`data/dataset/evaluation_dataset_200.csv`**: **[Raw Input Data]** This is the 200 random English-Vietnamese baseline pulled directly from the `opus100` parallel corpus. It acts as the "Gold Standard" Human Reference for grading.
- **`data/google_translate/`**: **[Intermediate Output]** Contains the CSV generated after running script `2a` (Google's NMT output).
- **`data/gemini_translate/`**: **[Intermediate Output]** Because Gemini strictly limits free tiers to 5 Requests-Per-Minute (RPM), the data is strategically chopped into disjointed chunk files (e.g., `translate_gemini_batch_1.csv`...). This design ensures that if the Python execution crashes due to connection loss, we do not lose our generated data.
- **`data/final_evaluation_dataset.csv`**: **[The Final Evaluation Benchmark]** This file is generated by `4_merge_data.py` and `5_classify_sentences.py`. It flawlessly stitches together the Human reference, Google translations, Gemini batched translations, and the linguistic "category" metadata into one massive dataframe. This is the **final aligned source** utilized by the evaluation scripts.

## 3. Data Pipeline: How to Run the Code
**System Requirements:** Python 3.10+
**Install Dependencies:**
```bash
pip install pandas datasets deep-translator python-dotenv google-genai sacrebleu matplotlib seaborn
```

**Execution Sequence:**
*(Run these scripts sequentially in your terminal)*
1. `python 1_prepare_data.py`: Fetches the dataset and saves `evaluation_dataset_200.csv`.
2. `python 2a_translate_google.py`: Rapidly translates the corpus using Google Translate binding.
3. `python 2b_translate_gemini.py`: Translates using Gemini. *Note: Deliberately executes slowly (enforces `time.sleep(15)`) to respect the 429 Resource Exhausted Limit. Employs fallback saves.*
4. `python 4_merge_data.py`: Aggregates the disconnected branches into a singular CSV dataframe.
5. `python 5_classify_sentences.py`: Contacts Gemini one last time to categorize the grammar structure (Simple, Complex, Idiom, Ambiguous).
6. `python 3_evaluate.py`: Calculates and logs the raw `sacrebleu` metrics.
7. `python 6_analyze_by_category.py`: Extracts the BLEU/chrF metrics divided by grammatical groupings.
8. `python 7_generate_charts.py`: Renders visual analytics directly into the `chart/` folder.

## 4. Evaluation Results & Interpretation
All algorithm calculations are executed by `sacrebleu`.
- **BLEU Score** mathematically maps exact token (n-gram) overlap.
- **chrF Score** mathematically calculates character string overlaps. Because Vietnamese heavily utilizes isolating and multi-compound phrasing, chrF inherently understands natural textual cohesion significantly better than BLEU point penalties.

**Final Computational Results:**
- **Google Translate:** BLEU 12.45 | chrF 31.24
- **Gemini (LLM):** BLEU 24.09 | chrF 43.21

**Qualitative Analytical Breakdown:**
- **Simple Sentences:** Gemini operates via implicit conversational memory (Discourse), enabling it to predict appropriate pronouns (e.g., mapping exclusive vs. inclusive plural pronouns) considerably better than SMT architectures.
- **Complex Clauses:** Mechanical NMT fails entirely to decode embedded relative clauses, rendering unintelligible overlaps in Vietnamese. LLM organically constructs the syntax tree anew, peaking at `51.14` chrF score.
- **Ambiguous Context:** Paradoxically, LLMs struggle violently in this regard due to **"Generative Hallucinations"**. Without sufficient context clues, the LLM hallucinates an imaginary narrative, straying entirely from the absolute literal definition where Google Translate excels.

## 5. Troubleshooting Runtime Errors
- **`ModuleNotFoundError: No module named 'cgi'`:** Occurs exclusively against Python 3.13 due to deprecated legacy modules inside the `googletrans` library. The pipeline strictly integrated `deep-translator` recently to effortlessly circumvent this exception.
- **`429 RESOURCE_EXHAUSTED` (Google API):** Indicates you violated the 5/RPM bottleneck. Script `2b_translate_gemini.py` inherently solves this: When detecting a server drop, it triggers a `time.sleep(60)` safety lock to refresh broadband, subsequently bypassing packet drops.
- **"Missing API Keys":** Rectified by instantiating a `.env` file at the root workspace directory with `gemini_key="YOUR_API_KEY"`.
